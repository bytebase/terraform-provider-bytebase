# Migrate from 3.16.3 to 3.17.0

## Breaking changes

### `branding_logo` removed from `workspace_profile` setting

The `branding_logo` field has been removed from the `workspace_profile` block in the `bytebase_setting` resource and data source. The branding logo is now managed via the new `bytebase_workspace` resource.

Before:

```terraform
resource "bytebase_setting" "profile" {
  name = "settings/WORKSPACE_PROFILE"

  workspace_profile {
    external_url = "https://bytebase.example.com"
    branding_logo = "data:image/png;base64,..."
  }
}
```

After:

```terraform
resource "bytebase_setting" "profile" {
  name = "settings/WORKSPACE_PROFILE"

  workspace_profile {
    external_url = "https://bytebase.example.com"
  }
}

resource "bytebase_workspace" "main" {
  logo = "data:image/png;base64,..."
}
```

### Classification `levels` schema changed

The `levels` block inside `classification` has changed:

- **Removed**: `id` (String) and `description` (String) fields.
- **Added**: `level` (Number) — the numeric level for ordering. Higher means more sensitive.

Before:

```terraform
resource "bytebase_setting" "classification" {
  name = "settings/DATA_CLASSIFICATION"

  classification {
    id    = "my-classification"
    title = "My Classification"

    levels {
      id          = "1"
      title       = "Level 1"
      description = "Low sensitivity"
    }
    levels {
      id          = "2"
      title       = "Level 2"
      description = "High sensitivity"
    }
  }
}
```

After:

```terraform
resource "bytebase_setting" "classification" {
  name = "settings/DATA_CLASSIFICATION"

  classification {
    id    = "my-classification"
    title = "My Classification"

    levels {
      title = "Level 1"
      level = 1
    }
    levels {
      title = "Level 2"
      level = 2
    }
  }
}
```

### Classification `classifications` schema changed

Inside the `classifications` block:

- **Removed**: `description` (String) field.
- **Changed**: `level` field type changed from `String` (level ID) to `Number` (numeric level matching `levels.level`).

Before:

```terraform
    classifications {
      id          = "1-1"
      title       = "User basic info"
      description = "Basic user information"
      level       = "2"
    }
```

After:

```terraform
    classifications {
      id    = "1-1"
      title = "User basic info"
      level = 2
    }
```

## Features

### New `bytebase_workspace` resource and data source

A new `bytebase_workspace` resource and data source are now available for managing workspace-level settings.

The **resource** supports:

- `title` — the workspace display name.
- `logo` — the branding logo (replaces `branding_logo` from workspace profile).
- `license` — upload a license key to activate a subscription plan (write-only, sensitive).
- `subscription` — computed block showing the current subscription status.

The **data source** exposes all read-only fields including subscription status.

```terraform
# Read workspace info and subscription status.
data "bytebase_workspace" "current" {}

output "workspace_plan" {
  value = data.bytebase_workspace.current.subscription[0].plan
}

# Manage workspace settings and license.
resource "bytebase_workspace" "main" {
  title   = "My Workspace"
  logo    = "data:image/png;base64,..."
  license = "your-license-key"
}

# The subscription block is computed after applying:
# subscription[0].plan         = "ENTERPRISE"
# subscription[0].seats        = 50
# subscription[0].instances    = 20
# subscription[0].expires_time = "2027-01-01T00:00:00Z"
# subscription[0].trialing     = false
```

### New `bytebase_idp` resource and data sources

A new `bytebase_idp` resource, `bytebase_idp` data source, and `bytebase_idp_list` data source are now available for managing SSO identity providers. Supports OAuth2, OIDC, and LDAP types.

```terraform
# OAuth2 example (GitHub)
resource "bytebase_idp" "github" {
  resource_id = "github"
  title       = "GitHub"
  domain      = "github.com"
  type        = "OAUTH2"

  oauth2_config {
    auth_url      = "https://github.com/login/oauth/authorize"
    token_url     = "https://github.com/login/oauth/access_token"
    user_info_url = "https://api.github.com/user"
    client_id     = "your-client-id"
    client_secret = "your-client-secret"
    scopes        = ["user"]

    field_mapping {
      identifier   = "email"
      display_name = "name"
    }
  }
}

# OIDC example (Okta)
resource "bytebase_idp" "okta" {
  resource_id = "okta"
  title       = "Okta"
  type        = "OIDC"

  oidc_config {
    issuer        = "https://example.okta.com"
    client_id     = "your-client-id"
    client_secret = "your-client-secret"
    scopes        = ["openid", "profile", "email"]

    field_mapping {
      identifier   = "email"
      display_name = "name"
    }
  }
}

# LDAP example
resource "bytebase_idp" "ldap" {
  resource_id = "company-ldap"
  title       = "Company LDAP"
  type        = "LDAP"

  ldap_config {
    host              = "ldap.example.com"
    port              = 636
    security_protocol = "LDAPS"
    bind_dn           = "cn=admin,dc=example,dc=com"
    bind_password     = "admin-password"
    base_dn           = "ou=users,dc=example,dc=com"
    user_filter       = "(uid=%s)"

    field_mapping {
      identifier   = "uid"
      display_name = "cn"
    }
  }
}

# List all identity providers
data "bytebase_idp_list" "all" {}

# Read a single identity provider
data "bytebase_idp" "github" {
  resource_id = "github"
}
```
