# Migrate from 3.15.2 to 3.16.1

## Breaking changes

### Parent field is now optional for workspace-scoped resources

The `parent = "workspaces/-"` is no longer needed. The `parent` field for workspace-scoped resources is now **optional** — simply remove it and the provider will automatically resolve the workspace.

Affected resources and data sources:

- `bytebase_iam_policy` (resource and data source)
- `bytebase_policy` (resource, data source, and list data source)
- `bytebase_service_account` (resource)
- `bytebase_workload_identity` (resource)
- `bytebase_service_account_list` (data source)
- `bytebase_workload_identity_list` (data source)
- `bytebase_database_list` (data source)

Before:

```terraform
resource "bytebase_iam_policy" "workspace_iam" {
  parent = "workspaces/-"

  iam_policy {
    binding {
      role    = "roles/workspaceAdmin"
      members = ["user:admin@example.com"]
    }
  }
}

resource "bytebase_service_account" "ci_bot" {
  parent             = "workspaces/-"
  service_account_id = "terraform"
  title              = "CI Bot"
}

resource "bytebase_policy" "query_data_policy" {
  parent = "workspaces/-"
  type   = "DATA_QUERY"
  query_data_policy {
    maximum_result_rows     = 100
    disable_export          = true
    disable_copy_data       = true
    allow_admin_data_source = false
  }
}
```

After:

```terraform
# parent defaults to workspace when not specified.
resource "bytebase_iam_policy" "workspace_iam" {
  iam_policy {
    binding {
      role    = "roles/workspaceAdmin"
      members = ["user:admin@example.com"]
    }
  }
}

# parent defaults to workspace when not specified.
resource "bytebase_service_account" "ci_bot" {
  service_account_id = "terraform"
  title              = "CI Bot"
}

# parent defaults to workspace when not specified.
resource "bytebase_policy" "query_data_policy" {
  type = "DATA_QUERY"
  query_data_policy {
    maximum_result_rows     = 100
    disable_export          = true
    disable_copy_data       = true
    allow_admin_data_source = false
  }
}
```

For project-level resources, you still need to specify the parent explicitly:

```terraform
resource "bytebase_iam_policy" "project_iam" {
  parent = "projects/sample"

  iam_policy {
    binding {
      role    = "roles/projectOwner"
      members = ["user:owner@example.com"]
    }
  }
}

resource "bytebase_service_account" "project_sa" {
  parent             = "projects/sample"
  service_account_id = "project-ci"
  title              = "Project CI Bot"
}
```

## Features

### Add `allow_just_in_time_access` to project

The `bytebase_project` resource now supports the `allow_just_in_time_access` field to control whether just-in-time access is allowed in the project.

```terraform
resource "bytebase_project" "sample" {
  resource_id             = "sample"
  title                   = "Sample Project"
  allow_just_in_time_access = true
}
```

### Add `REQUEST_ACCESS` approval source

The `workspace_approval` setting now supports a new `REQUEST_ACCESS` source for approval rules.

```terraform
resource "bytebase_setting" "approval" {
  name = "settings/WORKSPACE_APPROVAL"

  workspace_approval {
    rules {
      source    = "REQUEST_ACCESS"
      condition = "true"
      flow {
        # ...
      }
    }
  }
}
```
